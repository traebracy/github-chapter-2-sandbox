# Pull Request Practice

## What I Learned

This project helped me practice the basic GitHub pull request workflow. I learned how branches, commits, pushing, pull requests, merging, and cleanup all connect together when working on a repository.

## Skills Practiced

Through this practice, I learned how to:

* Create and switch to a new branch
* Make changes to a file
* Stage changes using `git add`
* Commit changes with a message
* Push a branch to GitHub
* Open a pull request
* Compare changes between branches
* Merge a pull request into the main branch
* Close pull requests that are no longer needed
* Delete unused branches after they are merged or no longer needed

## What I Understand Now

I understand that a branch is like a separate workspace where I can make changes without directly affecting the main version of the project. After I finish making changes, I can push the branch to GitHub and create a pull request.

A pull request is a way to review and compare changes before merging them into the main branch. Once the pull request is merged, the changes become part of the main project.

I also learned that closed or merged pull requests do not automatically delete branches. If a branch is no longer needed, it can be deleted separately to keep the repository clean.

## Commands I Practiced

```bash
git status
git branch
git checkout -b branch-name
git add .
git commit -m "commit message"
git push --set-upstream origin branch-name
```

## Reflection

At first, pull requests felt confusing because there were multiple parts involved: local branches, remote branches, commits, GitHub comparisons, and merging. After practicing the process, I understand that each step has a purpose.

This practice helped me become more comfortable with Git and GitHub. I now have a better understanding of how developers use branches and pull requests to manage changes safely and organize their work.
